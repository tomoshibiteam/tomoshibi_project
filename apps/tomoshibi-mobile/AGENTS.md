# AGENTS.md

## Purpose
このリポジトリでは、灯火（TOMOSHIBI）の事業・プロダクト前提の認識一致を維持する。

## Required Context
仕様提案、UX設計、機能追加、PRD作成、ピッチ素材作成の前に、以下を確認する。

1. `docs/business/TOMOSHIBI_COMMON_UNDERSTANDING.md`
2. `docs/business/COMMON_UNDERSTANDING_OPERATIONS.md`
3. `docs/business/DECISION_LOG.md`

## Output Rules
- 出力は「確定事項」と「仮説」を分けて記述する。
- 基準文書と矛盾する案を出す場合は、例外理由を明記する。
- 新しい重要判断が生じた場合は、`docs/business/DECISION_LOG.md` に追記案を含める。

## Change Rules
- 基準文書の意味を変更する提案を行う場合、変更理由と影響範囲を明記する。
- 文書更新時は、`最終更新日` を更新する。
